# Publication Checklist

Use this checklist before making the repository public.

- [ ] No `.env`, API keys, access tokens, cookies, logs, or private URLs.
- [ ] No private customer, company, project, or browser-history data.
- [ ] License file is present.
- [ ] README explains what the project does in the first screen.
- [ ] Install and quick-start commands work from a clean checkout.
- [ ] `maintainer-agent doctor` reports local setup status.
- [ ] Tests pass with `python -m unittest discover -s tests`.
- [ ] CI is configured for pull requests.
- [ ] README links to current CI status.
- [ ] Public maintenance log entries contain only public, non-sensitive data.
- [ ] Security reporting instructions are present.
- [ ] Example inputs are synthetic and safe to publish.
- [ ] GitHub description, topics, and first release are prepared.
